## 1.4.0 — 2026-07-16

### 新增

- Skill 新增“设计母版 → 实时截图 → 对照初版 → 迭代”的视觉闭环；结构校验或取色成功不再等同于皮肤完成
- Skill 发布包内附 `rose` / `portal` 初版视觉目标，脱离源码仓库执行时也能做同屏对照
- 一键生成支持显式 `--preset` 与 `--art-position`，可保留初版 `rose` / `portal` 的完整构图、层级和表面处理
- 新增设计母版选择参考：亮暖宽幅人物素材优先 `rose`，暗色电影感宽幅素材优先 `portal`，其余素材才使用 `adaptive`

### 修复

- 修复当前粉色宽幅人物图被误判为通用自适应主题、最终只呈现为淡化壁纸的问题
- 双击生成的本地兜底判断同步升级，不依赖 AI 时也能把典型素材路由到更接近初版的母版

### 说明

- 继续只通过 `127.0.0.1` CDP 热应用主题，不修改官方 `.app`、`app.asar`、代码签名或 API 配置

## 1.3.1 — 2026-07-16

### 修复

- 修复任意图片都被套入固定黑绿/青紫配色的问题：自定义主题现在会从图片缩略样本中提取主色、辅助色与视觉焦点，并使用独立 `adaptive` 预设
- 修复新版 Codex 任务 DOM 不再包含 `article` / `[role="main"]` 时正文玻璃层完全失效的问题；任务页改为在主工作面直接提供稳定的高可读遮罩
- 方图或竖图不再把高饱和素材直接铺到正文下方；任务页会自动强化降噪，首页仍保留清晰主题图
- 浅色壳不再把所有自定义图片强制配成粉色侧栏与控件，选中态、按钮和输入框会跟随图片强调色
- 修复非 `rose` 主题品牌缩略图按原始大图尺寸覆盖首页，以及横幅裁剪建议卡导致卡片不可见、不可点击的问题

### 改进

- Skill 验收新增任务阅读区视觉质量要求，避免“注入结构通过、实际效果很差”的假绿
- 首页验证新增品牌缩略图尺寸与建议卡真实点击命中检查；截图增加多帧合成器预热，降低瞬时黑色瓦片
- 图片分析完全在本机通过 `sips` 与已验证的 Codex 内置 Node 完成，不上传素材、不增加网络依赖

## 1.3.0 — 2026-07-16

### 新增

- 新增可由 Skill 或终端直接调用的 `generate-dream-skin-macos.sh`：单条命令完成安装/更新、图片制备、主题写入、应用与实时验证
- 新增 `Generate Codex Dream Skin.command` 双击入口；安装后也会在桌面创建独立的 Generate 启动器

### 改进

- `codex-dream-skin-studio` Skill 改为默认走一键生成流程，也可先用 AI 生成横向背景图再交给本机脚本
- 一键流程支持主题名、口号、三组配色、CDP 端口、验收截图、仅生成和跳过严格验证等参数
- Generate 桌面入口会优先复用当前主题会话的 CDP 端口；发布 ZIP 不再携带 `._*` AppleDouble 元数据文件
- 若 Codex 已运行但未开启经验证的本机 CDP，仍会明确请求一次重启确认；不会静默重启应用

### 说明

- 继续只使用 `127.0.0.1` CDP 注入，不修改官方 `.app`、`app.asar`、代码签名或 API 配置

## 1.2.0 — 2026-07-16

### 新增

- 新增可实际运行的粉系人物主题：使用独立纯横幅素材重构品牌栏、主页人物横幅、四张原生建议卡、输入区与右下拍立得装饰
- 主题配置新增 `preset` 字段；`rose` 预设会启用对应的响应式布局和视觉令牌

### 改进

- 四张主页卡片继续使用 Codex 原生按钮、原生图标和真实点击行为，只重绘视觉，不伪造任务或导航数据
- 粉系预设在主页、任务页和侧栏保持统一浅色玻璃质感，并遵守减少动态效果设置
- 注入器升级时会按版本重建装饰层，避免热更新继续复用旧版 DOM
- `doctor` 只验证与当前脚本路径及版本一致的实时会话，避免开发树误判已安装的旧版本
- 截图验收会先预热 Chromium 合成器再保存稳定帧，避免主题刚更新时出现瞬时黑色瓦片

### 说明

- README 图库仍是视觉方向合成图；实际皮肤保留当前 Codex 的真实侧栏、项目、任务和系统窗口框架
- CDP 仍只绑定本机回环地址，不修改官方应用、`app.asar` 或代码签名

## 1.1.0 — 2026-07-16
